# Daily Journal

Renders a single retrospective markdown file for today's session. Surveillance-only — no order placement, no position-sizing recommendations.

## Prerequisites

- The Tauri app (`pnpm tauri dev` or production build) must be running.
- TWS / IB Gateway must be connected and reachable; otherwise `ibkr_get_executions` returns empty and the Trades section degrades gracefully.

## Inputs

- **SQLite tracker DB** at `~/.local/share/com.quantyc.qqk/tracker.sqlite` (Linux). On macOS / Windows, resolve via Tauri's `app_local_data_dir()` for identifier `com.quantyc.qqk`.
- **`ibkr_get_executions(date: "YYYY-MM-DD")`** Tauri command, returning `Vec<IbkrExecution>` for today's ET date.

## Output

`journal/YYYY-MM-DD.md` (gitignored). Idempotent — overwrites on re-run.

## Procedure

Follow this checklist exactly. Do not invent fields the data doesn't have; prefer empty sections over fabricated reasoning.

1. **Compute today's ET date** as `YYYY-MM-DD`. Use the user's wall clock; convert to `America/New_York` if the host is in a different zone.

2. **Resolve the SQLite path.** Linux: `~/.local/share/com.quantyc.qqk/tracker.sqlite`. macOS: `~/Library/Application Support/com.quantyc.qqk/tracker.sqlite`. Windows: `%APPDATA%\com.quantyc.qqk\tracker.sqlite`.

3. **Read SQLite directly** (read-only, via `sqlite3` CLI in a Bash heredoc):

   - **Scanner highlights** — `tracked_tickers` rows where `added_at` falls within today's ET session AND `source = 'scanner'`. Columns: `symbol`, `tags`, `added_at`, `notes`.
   - **Detected setups** — `setups` rows where `detected_at` falls within today's ET session. Columns: `id`, `symbol`, `strategy`, `direction`, `trigger_price`, `stop_price`, `targets`, `raw_signals`, `thesis`, `thesis_json`, `status`.
   - **News context** (best-effort) — `news_cache` rows for symbols touched today, sorted by `published_at` desc.
   - **Agent journal sections** — `journal_entries` rows where `journal_date = '<YYYY-MM-DD ET>'`. Columns: `section`, `body_md`, `written_by`, `written_at`. The Phase-7 EOD review agent writes a row keyed on `(journal_date, section='EOD Review (Agent)')`; surface its `body_md` verbatim under that section heading. Other section names also render verbatim under their heading — sort by `written_at ASC` so multiple agent writes within a day appear in chronological order.

   ET-session window: `[YYYY-MM-DD 04:00 ET, YYYY-MM-DD+1 04:00 ET)` covers premarket through after-hours. Express bounds in UTC ISO-8601 for the SQLite comparison.

4. **Fetch today's fills.** Invoke `ibkr_get_executions` with today's ET date string. The command requires the app + TWS/Gateway running. If it errors or returns `[]`, continue rendering with the Trades section reading "No fills today."

5. **Match fills to setups.** For each fill, scan `setups` for the same symbol with `detected_at` within the prior 5 trading days. If a match exists, render the linked setup's `thesis` (markdown) and structured `trigger_price` / `stop_price` / `targets` beneath the trade row. If no match, render the trade with a `_no detected setup_` flag — do not fabricate reasoning.

6. **Render** `journal/YYYY-MM-DD.md` with the template below. Create the `journal/` directory if missing.

## Template

See `references/journal-template.md` in the skill bundle, or use this inline template:

```
# Trading Journal — {YYYY-MM-DD}

_Generated by /journal at {ISO-8601 timestamp ET}_

## Market Overview

_Skim SPY / QQQ / IWM / VIX from the most recent close in bars_cache if present; otherwise note "(no index bars cached)". Keep this section to 3–4 lines._

## Scanner Highlights

- **{symbol}** — added at {HH:MM ET}; tags: `{tags}`{; notes: "{notes}" if present}

_If no rows: "No scanner highlights today."_

## Detected Setups

### {symbol} — {strategy} ({direction})

- **Trigger:** {trigger_price} | **Stop:** {stop_price} | **Targets:** {targets}
- **Status:** {status}
- **Raw signals:** {raw_signals_json}
- **Thesis:** {thesis or "(no thesis generated)"}

_If no setups: "No setups detected today."_

## Trades

### {symbol} — {side} {qty} @ {avg_price} ({HH:MM:SS ET})

- **Order ID:** {order_id} | **Exec ID:** {exec_id}
- **Linked setup:** {setup id + strategy if matched, else "no detected setup"}
- **Setup parameters:** {trigger / stop / targets if linked}
- **Thesis at detection:** {linked setup's thesis, or omit if unlinked}

_If no fills: "No fills today."_

## EOD Review (Agent)

_If a `journal_entries` row exists for `(journal_date=today, section='EOD Review (Agent)')`, render its `body_md` verbatim here. If no row exists, render: "No agent EOD review for today (yet) — see `agent/eod_review.py`."_

## Notes

_Reserved for the user. Do not auto-populate._
```

For any other agent-authored section — e.g. an interactive Claude
Code session writing a free-form section name — render it as its
own `## {section}` block with the body verbatim, ordered after
"EOD Review (Agent)" and before "Notes".

## Edge cases

- **Weekends / holidays:** the SQLite queries and `ibkr_get_executions` will return empty; render the file with empty sections rather than erroring.
- **TWS not running:** `ibkr_get_executions` will error with a connection message. Capture the error, render the Trades section as "_TWS/Gateway unavailable: {err}_", and continue.
- **Re-running:** overwrite `journal/YYYY-MM-DD.md` in place; do not append.
- **Multi-account:** v1 surfaces all fills returned by the command without account aggregation. The IBKR adapter is single-account today.
